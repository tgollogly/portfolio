Case study

How it's built — and why

The problem

People representing themselves in court struggle to assemble a proper evidence bundle — a title page, a clear index, and a labelled divider for every document. I wanted to make building that structure a five-minute job.

Stack & why I chose it

Vanilla JavaScript plus the browser's own print engine (window.print() with a dedicated print stylesheet) to produce the PDF — no PDF library needed, which keeps it lightweight, and the documents stay entirely on the user's device, which matters for legal material.

Key decisions

Reordering works by swapping array items and re-rendering; a separate print layout builds the index table and one tab-divider page per document with proper page breaks. I intentionally avoided localStorage so it behaves consistently everywhere and stays session-only.

Trade-offs & what's next

It organises and indexes the documents rather than merging the actual PDF files. Next: drag-and-drop reordering and true PDF merging.
